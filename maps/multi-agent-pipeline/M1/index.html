M1

System Shape

M1 scopes the workflow enough to choose the track: Unknown / Scope First, Single-Agent / APS, or Multi-Agent / MAPS. It protects the project from premature overbuilding by using Research and Recommend plus Agentic Systems Program Manager guidance from M0 when that role exists.

Shape the system Back to M0 Foundation Enter A0 if APS

M1 output

The required artifact is `system-shape.md`, including the S/M/U decision and next-track recommendation.

Skill

/shape asks the first S/M/U question, then uses Research and Recommend to propose the track.

Template

templates/system-shape.md captures the track decision.

Primary handoff

If S, enter APS. If M, continue to M2 Roster. If U, keep scoping until the operating model is visible.

ASPM guidance

If M0 defines or references an Agentic Systems Program Manager role, `/shape` uses that guidance for phase boundaries, memory/RAG implications, and next-skill routing.

M0 handoff

M1 receives the foundation artifact and turns it into a system-track decision.

Foundation input

Read `project-foundation.md`, the source inventory, EventStorming Lite notes, and the open foundation questions before choosing a track.

Memory input

Check `.maps/foundation-preferences.json`, `.maps/rag-updates.json`, and the named foundation run note for configured notes, memory, and RAG locations.

Shape output

Record the S/M/U recommendation in `system-shape.md` and write the `/shape` run note through the shared MAPS memory helper.

Research sources

Use workflow-shape, service-blueprint, team-topology, role-pressure, and agent-orchestration references before recommending the track.

python "$CODEX_HOME/skills/foundation/scripts/maps_memory.py" complete-run \
  --project . \
  --skill /shape \
  --phase M1 \
  --output system-shape.md \
  --summary-file system-shape.md \
  --memory-updates "Track decision captured"

S/M/U decision

Start with uncertainty when the developer does not know the answer yet.

Unknown / Scope First

Use when workflow, actors, data, approvals, handoffs, risks, or failure modes are not clear enough yet.

Single-Agent / APS

Use when one coherent agent can own the outcome with one tool/memory context and low handoff burden.

Multi-Agent / MAPS

Use when distinct roles, tools, permissions, memory, review, escalation, or parallel work are justified.

Recommendation standard

Every M1 recommendation should include confidence, evidence, contrary signals, what would change the decision, any M0-defined ASPM guidance, and the next skill to run.

System Shape depends on workflow mapping, role-pressure checks, and handoff/orchestration references.
