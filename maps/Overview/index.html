What MAPS gives builders

The framework keeps phase work, reusable skills, templates, catalogs, and repo structure connected.

Foundation preflight

M0 asks memory first: whether RAG exists, where notes live, how to access each store, and which locations are canonical.

Conversational skills

Every MAPS skill asks one question at a time, waits for the answer, and only writes artifacts after missing decisions are clear.

Clear handoffs

Every skill ends with a completion report: status, outcome, memory update, and the recommended next skill to run.

Reusable skills

Each phase can be executed by compatible agent tooling through `SKILL.md` workflows and helper scripts.

Training workflow

Teach MAPS as a resettable operating loop: ask first, write the foundation, inspect memory, then continue.

1. Run M0 preflight

Start by asking whether the project has RAG. If not, ask where notes live. Ask one question at a time, then capture locations, access method, and canonical-store policy before project identity.

2. Inspect the artifact

Review `project-foundation.md`, `.maps/foundation-preferences.json`, `.maps/rag-updates.json`, and `notes/maps-runs/foundation.md` before M1.

3. Read the handoff

At completion, read the status, outcome, memory update, and next-skill recommendation before moving forward.

4. Reset when wrong

Use `/foundation --wipe` to preview cleanup. Confirm only when the listed files are the MAPS-created foundation artifacts you want removed.
