Org Chart

Roles versus Agents

Roles describe responsibility, voice, maturity, and authority. Role+ adds bounded automation without agent status. Agents are implemented systems with state, tools, memory, policies, evaluations, handoffs, and stop conditions.

Next: role maturity Back to org chart

Why the distinction matters

A role describes organizational responsibility. Role+ means the role has bounded automation, such as a heartbeat check, but no independent runtime state. An agent is an implemented operating system for that role. MAPS keeps those separate so a title or timer never accidentally grants tools, memory writes, production access, or autonomy.

Role

A role is a named organizational seat with purpose, voice, responsibilities, maturity, authority boundaries, and evidence expectations.

Role+

Role+ has automation enabled, but no independent authority, no goal outside the automation rules, and no runtime state. A memory file alone does not make it agentic.

Implemented agent

An agent has a goal, state, policy, tools, memory, evaluations, handoffs, escalation path, stop conditions, and an approved build path.

MAPS rule

Creating a role contract or adding Role+ automation does not automatically create or authorize an agent. Agent build status requires explicit approval, a build-ready design, declared tools, memory boundaries, evaluations, handoffs, escalation, and stop conditions.
