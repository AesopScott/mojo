Authority Taxonomy

Authority must be declared explicitly. A role title never grants broad permission by itself.

Next: operating cadence Back: role maturity

Authority levels

Choose the highest allowed level, then restrict it by domain, approval gate, and special declaration.

LevelNameMeaning
A0NoneNo role authority. Persona, voice, or context only.
A1ObserveCan read allowed context and summarize state. Cannot recommend action.
A2AdviseCan provide perspective, critique, risks, and options.
A3RecommendCan recommend decisions or next actions with rationale and evidence.
A4DraftCan draft artifacts, plans, messages, specs, tickets, or code for review.
A5CoordinateCan organize work, maintain checklists, route handoffs, and request inputs.
A6Execute With ApprovalCan prepare and execute actions only after explicit human or authorized-role approval.
A7Execute Within PolicyCan execute pre-approved actions inside named rules, limits, and rollback paths.
A8ApproveCan approve specified decisions or artifacts inside a named domain.
A9VetoCan block specified decisions, releases, actions, or escalations inside a named domain.
A10Autonomous Within BoundsCan plan, act, observe, update memory, and continue inside explicit boundaries and stop conditions.
A11Emergency OnlyCan exceed normal authority only under declared incident or safety conditions.
A12OwnerOwns the outcome, operating loop, review rhythm, and improvement backlog for a domain.

Role capability checklist

Use this as the explicit inventory of what a role could do. Anything checked must still be constrained by authority level, domain, approval gate, and revocation rule.

Advice and judgment

  • Read allowed context
  • Summarize state
  • Provide critique
  • Identify risks
  • Generate options
  • Recommend decisions
  • Recommend next actions
  • Explain rationale and evidence

Artifact work

  • Draft plans
  • Draft messages
  • Draft specs
  • Draft tickets
  • Draft code
  • Draft policies
  • Update checklists
  • Prepare review packages

Workflow ownership

  • Organize work
  • Maintain operating cadence
  • Request inputs
  • Route handoffs
  • Track blockers
  • Escalate stale work
  • Own review rhythm
  • Maintain improvement backlog

Tool and system use

  • Use named tools
  • Call APIs
  • Run scripts
  • Use MCP servers
  • Use connectors
  • Open or update tickets
  • Operate dashboards
  • Trigger automations

Memory, notes, and RAG

  • Read memory
  • Read notes
  • Read RAG results
  • Read source stores
  • Propose memory updates
  • Write memory updates
  • Write RAG/source updates
  • Log run evidence

Data and evidence

  • Collect sources
  • Validate evidence
  • Handle customer data
  • Classify sensitive data
  • Maintain provenance
  • Update evidence index
  • Reject unsupported claims
  • Flag data risk

Communication

  • Communicate internally
  • Draft external communication
  • Send external communication
  • Contact customers
  • Contact vendors
  • Publish public content
  • Represent the organization
  • Enforce no-external-communication

Money and commitments

  • Estimate cost
  • Spend within budget
  • Procure tools
  • Commit resources
  • Approve vendor work
  • Negotiate terms
  • Track token/time/resource limits
  • Escalate budget exceptions

Governance and risk

  • Interpret policy
  • Create governance proposals
  • Approve policy changes
  • Veto risky actions
  • Run compliance checks

People and roles

  • Recommend roles

  • Assign responsibility

Production and operations

  • Deploy changes

Authority controls

  • Act with human approval

  • read-only

  • human-approval-required
  • role-approval-required
  • two-key-approval
  • policy-bound
  • budget-bound
  • time-bound
  • scope-bound
  • memory-read

  • memory-propose

  • no-external-communication
  • tool-use
  • no-tool-use
  • production-access
  • no-production-access
  • veto-right
  • escalation-right
  • emergency-break-glass
  • self-improvement-propose
  • self-improvement-write
  • revocable

Required authority output

Every role contract must state highest authority level, authority by domain, special declarations, approval gates, evidence requirements, forbidden actions, escalation path, revocation or rollback path, and review cadence.
