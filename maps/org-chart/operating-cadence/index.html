Operating Cadence

Operating cadence defines when roles appear, what triggers them, and how implemented agents run visible loops.

Back to org chart Next: role handoffs Back: authority taxonomy

Role trigger types

A role should not appear randomly. It should activate because a named trigger matched its engagement rule.

Explicit invocation

A person calls the role by name or asks for its judgment.

Session start

A project hook loads the role when the project context requires it.

Phase boundary

The role appears when MAPS crosses a phase, approval gate, or artifact handoff.

Review gate

The role reviews a plan, artifact, release, policy, or role contract before action.

Event or signal

A webhook, queue, incident, metric, backlog threshold, or memory update triggers the role.

Scheduled cadence

The role runs on a daily, weekly, milestone, release, or governance rhythm.

How agents operate

An implemented agent should run a visible loop, not a hidden personality trick.

  1. Intake: read the trigger, role contract, allowed context, and current state.
  2. Plan: choose the next action inside authority, approval, and stop conditions.
  3. Act: draft, recommend, coordinate, call tools, or execute only inside the declared boundary.
  4. Record: write run notes, decisions, evidence, and memory updates to the approved locations.
  5. Report: state what happened, what changed, what remains open, and what approval is needed.
  6. Stop or continue: stop on completion, uncertainty, forbidden action, approval need, risk, budget limit, or escalation condition.

Cadence ladder

Roles can start light and become more operational as evidence and trust increase.

CadenceBehaviorTypical implementation
Passive referenceProvides context when consulted.Markdown role contract or memory note.
AdvisorGives first-person critique, options, and recommendations.Skill or loaded project instruction.
Review gateMust review named artifacts before approval.Checklist, hook, or required review step.
Workflow ownerMaintains rhythm, handoffs, blockers, and progress evidence.Skill plus helper script and run log.
OperatorExecutes approved tasks with tools.Script, MCP workflow, or tool-backed skill.
Autonomous loopPlans, acts, observes, remembers, and continues within explicit bounds.Active process, scheduled worker, or agent runtime.
Escalation authorityApproves, vetoes, or breaks glass in a named domain.Governance rule plus audit log.
