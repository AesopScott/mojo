MAPS Org Chart

This chart is a set of role agents meant to run an AI agency organization. The same structure can be adapted to run other kinds of organizations by changing the roles, authority boundaries, tools, memory, and operating cadence.

Legend

Use color to track role operating state. Red items do not exist yet. Blue items are active operators. Green items coordinate across roles, channels, or process lanes. Gold is reserved for approved executor agents with runtime proof and bounded autonomous authority.

Red

Role does not exist.

Blue

Operator: activated role with bounded monitoring or action authority.

Green

Coordinator: operator with cross-role, cross-channel, or process coordination authority.

Gold

Executor: approved agent runtime with tools, state, eval proof, audit trail, and escalation boundaries.

Typical company org chart

All names start red. Activated operators turn blue. Roles with approved coordination authority turn green. Executor agents turn gold only after runtime proof and explicit approval. Current role status comes from the Ana-owned roles directory in Obsidian.

How this becomes MAPS work

Each blue operator can be qualified by Ana and reviewed by Vik. Green coordinators carry approved cross-role or process authority. Gold executors expose activation, autonomy, authority, tool, hook, memory, and evaluation controls.

Role contract

Use /role with Ana to define the role voice, responsibility boundary, authority, engagement, and proof expectations.

Engagement

Decide whether the role is passive reference, advisor, review gate, workflow owner, operator, autonomous loop, or escalation authority.

Agent profile

Once a role becomes a gold executor, use the profile page to review activation, autonomy, authority, tools, hooks, memory, eval proof, and next MAPS phase.

Role handoff

Turn architecture into a build-ready packet with owner, scope, authority gates, acceptance criteria, verification, and memory updates.
