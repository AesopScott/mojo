MAPS Org Chart

This chart is a set of role agents meant to run an AI agency organization. The same structure can be adapted to run other kinds of organizations by changing the roles, authority boundaries, tools, memory, and operating cadence.

How this becomes MAPS work

Each role can move only when its source contract, autonomy assessment, eval proof, gates, and explicit approvals agree. Color is a training signal, not an authority grant.

Role contract

Use /role with Ana to define the role voice, responsibility boundary, authority, engagement, and proof expectations.

Engagement

Decide whether the role is passive reference, advisor, review gate, workflow owner, autonomous loop, or escalation authority.

Agent profile

Once a role becomes gold, use the profile page to review activation, autonomy, authority, tools, hooks, memory, eval proof, and next MAPS phase.

Role handoff

Turn architecture into a build-ready packet with owner, scope, authority gates, acceptance criteria, verification, and memory updates.
