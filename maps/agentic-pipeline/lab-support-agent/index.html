Lab 3

Support Agent

Build an agent that triages customer support tickets, drafts safe replies from approved knowledge, and escalates edge cases with useful context.

Ticket triage Knowledge retrieval Reply drafting Escalation rules Tone control Resolution evidence
Start the lab Previous lab Back to MAPS

Customer story

Harbor Health sells remote patient monitoring kits to clinics. Its support queue spikes after device onboarding, and agents spend too much time rewriting the same troubleshooting replies while missing urgent escalation signals.

User

Leah, the support lead, wants draft replies and triage labels before her team starts the morning queue.

Job

Classify the ticket, find approved help-center guidance, draft a reply in brand tone, and escalate anything medical, billing-sensitive, angry, or blocked.

Outcome

Support agents respond faster while still owning final send decisions and sensitive escalations.

Requirements

R1

Ticket labels

Classify tickets as onboarding, device offline, data sync, account access, billing, clinical concern, complaint, or unknown.

R2

Knowledge boundary

Use only approved help-center articles, macro snippets, device setup docs, and support policy notes.

R3

Escalation

Escalate clinical advice, refunds over policy, angry customers, repeated failures, VIP clinics, and missing knowledge articles.

R4

Draft reply

Include empathy, likely issue, steps, source article, what the human should verify, and a clear next action.

R5

Do not send

The agent drafts only. A human support agent must review and send every customer-facing response.

R6

Evidence

Save the ticket summary, selected label, retrieved source, draft reply, escalation decision, and confidence.

MAPS path

Use APS to build a support agent that is useful without crossing sensitive boundaries.

A0

Align

Create `harbor-support-agent`, ticket samples, approved knowledge folders, and demo boundaries.

A1

Define

Write the support lead brief with draft-only authority and escalation triggers.

A2

Design

Design triage, retrieval, reply drafting, confidence, and human approval workflow.

A3

Build

Implement the first slice: classify one ticket and draft a reply from one approved article.

A4

Equip

Add local knowledge retrieval, ticket fixtures, output logs, and optional help desk connector notes.

A5

Evaluate

Test clinical-advice refusal, billing escalation, missing article behavior, and tone consistency.

A6-A8

Ship and improve

Package the demo, observe review friction, and improve labels, tone, and escalation precision.

Developer deliverables

Agent artifacts

Brief, design, backlog, ticket fixtures, knowledge notes, and at least one draft response output.

Proof

One passing draft-quality eval and one escalation-boundary eval.

Demo

A short walkthrough showing ticket input, retrieved article, draft reply, confidence, and escalation decision.
