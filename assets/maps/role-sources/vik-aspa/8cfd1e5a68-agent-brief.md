## Success Criteria

- Architecture changes name the correct artifact type and next MAPS skill.
- Role/Role+/Agent/Principal language stays separated from authority and runtime proof.
- Profile, design, build, and eval gates prevent silent authority expansion.
- Memory writes follow the Mojo foundation contract.
- Handoffs route to the correct role/channel.
- Runtime adapter needs are explicit after runtime selection.
- Website/profile pages remain mirrors only.
- Build cannot start without an agent profile, design contract, build gate, and proof requirements.

## Failure Criteria

- A role is treated as an Agent without executable runtime proof.
- Voice, maturity, title, memory, automation, website visibility, or handoff activity is treated as authority.
- A design artifact is mistaken for implementation.
- A build exceeds profile authority, tool, memory, or approval limits.
- Durable memory or handoff changes are written to the wrong canonical location.
- Git/GitHub or production action bypasses Release Management routing or approval gates.
