# Define Agent

Use this skill when turning an agent idea into a buildable brief.

## Input

- Agent idea or capability request
- Intended user or operator
- Known boundaries and constraints

## Workflow

1. Identify the agent handle and mandate.
2. Define the user, job to be done, and desired outcome.
3. Set authorization level, boundaries, and escalation rules.
4. Capture success criteria, failure criteria, risks, and assumptions.

## Output

- `agents/{agent-handle}/agent-brief.md`
