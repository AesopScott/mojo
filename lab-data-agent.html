Lab 2

Data Agent

Build an agent that answers operational data questions from approved files, explains its calculations, and refuses unsafe or unsupported requests.

Customer story

FreshFork Foods delivers prepared meals to offices. Operations leaders need fast answers from weekly order exports, but the team keeps losing time in spreadsheets and arguing about metric definitions.

User

Andre, the operations director, asks questions about late deliveries, refunds, subscription churn, and region-level volume.

Job

Load approved CSV files, validate columns, calculate requested metrics, explain the formula, and show the rows or filters used.

Outcome

Andre can make staffing and route decisions without waiting for an analyst, while still seeing the evidence behind the answer.

Requirements

R1

Approved inputs

Use only files placed in `data/approved/`: `orders.csv`, `refunds.csv`, `customers.csv`, and `delivery-events.csv`.

R2

Metric contract

Maintain a `metrics.md` file defining late delivery rate, refund rate, churn risk, average order value, and active customer count.

R3

Answer format

Return answer, formula, filters, caveats, data quality warnings, and suggested next question.

R4

Privacy boundary

Never expose full customer addresses, payment fields, or individual customer histories unless the user asks for an approved support lookup.

R5

Uncertainty

Warn when columns are missing, dates are ambiguous, sample size is small, or source files are stale.

R6

Proof

For every numeric answer, include a reproducible calculation trace or saved query artifact.

MAPS path

Use APS to move from data question to working, evaluated agent.

A0

Align

Create `freshfork-data-agent`, folders, data boundaries, and lab sample files.

A1

Define

Write authority rules for approved files, metric definitions, and privacy refusals.

A2

Design

Design ingestion, validation, metric lookup, calculation trace, and answer format.

A3

Build

Implement the first slice: load one CSV and answer late delivery rate with formula and filters.

A4

Equip

Add local file access, schema checks, calculation scripts, and a saved output folder.

A5

Evaluate

Test missing columns, stale files, privacy-sensitive prompts, and wrong metric names.

A6-A8

Ship and improve

Package the class demo, observe confusing answers, and improve metric clarity.

Student deliverables

Agent artifacts

Brief, design, backlog, metric contract, and at least one working query output.

Proof

One passing metric calculation eval and one privacy-boundary refusal eval.

Demo

A short walkthrough showing data input, validation warning, final answer, and calculation trace.
