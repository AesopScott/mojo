What MAPS gives builders

The framework keeps phase work, reusable skills, templates, catalogs, and repo structure connected.

Phase structure

Every phase has a purpose, output, and place in the agent system lifecycle.

Reusable skills

Resource catalogs

Repos, tools, templates, and skills are attached to the phase where they create leverage.

Self-guided build path

01

Orient

Separate the multi-agent project scaffold from the agent development pipeline.

Check your understanding

M defines the multi-agent system. APS develops one agent or agent capability.

02

Start at Phase 0

Use Phase Alignment to confirm names, boundaries, files, and the resource structure.

Expected output

A MAPS scaffold with phase pages, skills, templates, catalogs, and a shared vocabulary.

03

Move through APS

Open each available phase page, use its skill and template, then check the output.

Before moving on

The phase output should be specific enough for another builder or agent to continue.

04

Connect the layers

Use the M scaffold to coordinate the agents that APS defines, designs, builds, and improves.
