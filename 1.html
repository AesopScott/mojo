Phase 1

Define

Define turns an agent idea into a usable brief. The brief identifies the agent job, user or operator, desired outcome, boundaries, success criteria, failure criteria, escalation points, risks, and assumptions.

Build the brief Back to Phase 0

Build navigation

The agent brief is the contract.

A strong brief makes later design and evaluation possible because it states exactly what the agent owns.

Inputs

An agent idea, workflow need, user request, or multi-agent role candidate.

Work

Clarify scope, non-goals, observable success, failure criteria, and human escalation points.

Output

A filled `agent-brief.md` ready to move into the Design phase.

Agent brief fields

These are the minimum fields the next phase needs.

Name

Short identifier for the agent or role.

User or operator

The person or system that uses, reviews, or supervises the agent.

Job to be done

The specific outcome the agent is responsible for.

Boundaries

In scope, out of scope, and unclear territory.

Success and failure

Observable criteria for done and explicit conditions that count as failure.

Risks and assumptions

What must be controlled, escalated, or proven later.
