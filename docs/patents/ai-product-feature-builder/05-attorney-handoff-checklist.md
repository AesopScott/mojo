# Attorney Handoff Checklist

## Goal Of Attorney Meeting

Ask whether a provisional patent application should be filed around Mojo's multi-stage, human-gated AI product and feature generation workflow.

## Materials To Bring

- This packet.
- Any diagrams created from the process spec.
- Screens or mockups if available.
- Notes showing when Scott conceived the process.
- Any public or private documentation that should establish development history.
- Prior-art links listed in `04-prior-art-notes.md`.

## Core Question For Counsel

Can we pursue claims around the staged workflow, not generic AI code generation?

Specifically:

- two intake modes;
- overview plan;
- requester approval;
- provider approval;
- detailed implementation plan;
- controlled AI build;
- validation package;
- human release gate;
- scoped deployment.

## Questions To Ask Counsel

1. Does the staged requester/provider approval workflow appear meaningfully distinct from the closest prior art?
2. Should the provisional focus on method claims, system claims, or both?
3. Should the application include new product requests and existing-product feature requests in one filing?
4. How should we describe AI without locking the claims to a specific model, vendor, or implementation?
5. How should we handle customer-submitted ideas that become outputs of the system?
6. Should the provisional include inventorship/audit-record features?
7. Should deployment scope be a major claim element?
8. Should the claims include marketplace/module reuse?
9. What additional prior-art search should be done before filing?
10. What diagrams would improve the application?

## Decisions To Make

- File provisional or wait.
- File one broad provisional or multiple narrower provisionals.
- Include both new-product and feature-request modes.
- Include implementation details now or keep them broad.
- Whether to claim audit records and approval records.
- Whether to claim tenant/customer-specific deployment.
- Whether to claim reusable module/marketplace promotion.

## Provisional Drafting Notes

Make sure counsel understands:

- Scott is the inventor of the process/system.
- AI and engineers may be builders, but they are not the inventors of this patent concept.
- Customer feature ideas are outputs/inputs within the system and are separate from inventorship of the process.
- The novelty should be argued around the workflow and gates.

## Documents Counsel May Draft

- Provisional patent application.
- Invention disclosure confirmation.
- Assignment paperwork, if needed.
- Follow-up claim strategy memo.
- Prior-art search report.

## Red Flags

- Claims that read like generic AI code generation.
- Claims that rely only on "human approval."
- Claims that omit the two-stage planning process.
- Claims that omit the provider/Mojo approval gate.
- Claims that fail to distinguish new-product requests from existing-product feature requests.
