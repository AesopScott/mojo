# Saturday Patent Session Agenda

Date: Saturday, May 30, 2026

## Objective

Prepare enough detail to decide whether to file a provisional patent application and give patent counsel a clear, specific invention package.

## 1. Confirm The Invention

Working statement:

Mojo has invented a multi-stage approval system for AI-assisted generation of software products and product features from user requests.

Confirm:

- The invention is the process/system.
- Scott is the inventor of that process/system.
- Individual generated products or features are outputs of the process.

## 2. Pick The Final Title

Candidate:

Multi-Stage Approval System for AI-Assisted Generation of Software Products and Product Features from User Requests

Alternatives:

- Human-Gated AI System for Generating Software Products and Product Features from Natural-Language Requests
- AI-Orchestrated Product and Feature Development System With Requester and Provider Approval Gates
- Governed AI Software Generation System for Customer-Requested Products and Features

## 3. Walk The Process End To End

Review:

- Request intake.
- Request classification.
- Overview plan.
- Requester approval.
- Mojo/provider approval.
- Detailed plan.
- Controlled AI build.
- Validation.
- Review package.
- Release gate.
- Deployment.
- Audit record.

## 4. Identify Must-Have Claim Elements

Mark each as essential or optional:

- Two intake modes.
- Overview plan.
- Requester approval.
- Provider approval.
- Detailed implementation plan.
- AI build environment.
- Validation.
- Review package.
- Human release gate.
- Deployment scope.
- Audit record.
- Marketplace/module reuse.

## 5. Review Prior Art

Focus on:

- US20250061402A1.
- US20250165890A1.
- US20240411674A1.
- US20250315223A1.
- US8365138B2.
- US20250328343A1.

Decision:

- Does the Mojo workflow still feel distinct enough to ask counsel about a provisional?

## 6. Draw Diagrams

Create at least:

- One high-level flowchart.
- One two-mode request diagram.
- One approval-gate diagram.
- One architecture diagram.

## 7. Counsel Questions

Prepare to ask:

- Can this be claimed as a specific governed workflow?
- Are the two approval gates enough to distinguish prior art?
- Should new-product and feature-request workflows be together or separate?
- What should be added to strengthen eligibility and novelty?

## 8. Filing Decision

Choose one:

- Proceed with provisional.
- Do more prior-art searching first.
- Build MVP first and document implementation details.
- Split into multiple inventions.

## 9. MVP Notes

If building before or alongside filing, the likely MVP is:

- In-product request button.
- AI-generated overview plan.
- Requester approval.
- Mojo approval dashboard.
- AI-generated detailed implementation plan.
- GitHub issue/branch or equivalent implementation package.
- Human release approval.

## 10. Output From Saturday

By the end of the session, produce:

- Final invention title.
- Final one-paragraph invention summary.
- Essential claim elements.
- Optional claim elements.
- Attorney questions.
- Filing recommendation.
- MVP notes.
